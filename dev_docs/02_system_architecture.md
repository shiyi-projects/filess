# 系统架构

## 1. 架构目标

本架构用于在“本地文件安全”和“云端 AI 分类能力”之间建立清晰边界。系统必须做到：

1. AI 只负责建议，不直接控制文件系统。
2. 本地规则层负责安全校验、执行和回滚。
3. 搜索、审计和 RAG 都有明确的本地存储来源。

## 2. 总体拓扑

系统由四个运行单元组成：

1. `React + TypeScript`
   负责 UI、状态展示、人工确认、搜索交互、设置管理。
2. `Tauri + Rust`
   负责前后端桥接、任务编排、文件系统操作、本地搜索查询、SQLite 访问、Sidecar 调用。
3. `Python Sidecar`
   负责类型识别、内容提取、特征构建、RAG 检索、硅基流动 API 调用、向量写入。
4. `本地存储`
   SQLite 存放业务数据与搜索索引，ChromaDB 存放向量记忆。

## 3. 模块边界

### 3.1 UI 层

职责：

1. 接收拖拽路径
2. 展示批次状态和任务详情
3. 发起搜索
4. 展示搜索结果与本地路径
5. 执行打开文件、定位目录、复制路径
6. 展示人工确认弹窗
7. 编辑和保存设置

禁止职责：

1. 不直接访问文件系统
2. 不直接调用硅基流动 API
3. 不直接写 SQLite 或 ChromaDB

### 3.2 Rust 编排层

职责：

1. 接收 UI 命令并维护应用状态
2. 将拖拽内容切分为批次与任务
3. 控制解析并发和模型请求节流
4. 调用 Python Sidecar
5. 执行策略校验
6. 执行移动、重命名、撤销
7. 读写 SQLite
8. 对外暴露 Tauri 命令

禁止职责：

1. 不自行实现文档解析
2. 不直接内嵌大模型推理

### 3.3 Python Sidecar

职责：

1. 识别文件类型
2. 提取元数据和内容片段
3. 对文件夹进行浅采样
4. 构建标准化特征
5. 执行 Chroma 检索
6. 调用硅基流动聊天模型与 embedding 模型
7. 将符合条件的样本写入向量库

禁止职责：

1. 不执行任何文件移动或重命名
2. 不决定最终物理路径
3. 不跳过 Rust 的策略安全校验

### 3.4 策略安全层

策略安全层逻辑运行在 Rust 内部，职责包括：

1. 校验 `category_id` 是否存在于分类树
2. 根据分类树拼装受控目标路径
3. 校验目标路径必须位于目标根目录下
4. 处理非法字符与目录层级限制
5. 判断是否需要人工确认
6. 决定重名处理策略

## 4. 进程模型

首版采用 `单桌面主进程 + 单 Python Sidecar` 模式。

### 4.1 Sidecar 启动方式

1. 应用启动时由 Tauri 拉起 Python Sidecar。
2. Rust 保存 Sidecar 进程句柄和健康状态。
3. Sidecar 异常退出时，Rust 记录错误并将新任务标记为失败。

### 4.2 Sidecar 通信方式

1. Rust 与 Python Sidecar 使用 `JSON-RPC over stdio`。
2. 每次请求包含 `request_id`、`method`、`params`。
3. Sidecar 必须返回结构化结果或结构化错误。
4. 不使用本地 HTTP 服务，避免额外端口与防火墙问题。

## 5. 核心调用链

### 5.1 自动整理链路

1. UI 拖入文件或文件夹
2. Rust 创建批次和任务
3. Rust 调用 Sidecar 解析对象
4. Sidecar 返回标准化特征
5. Sidecar 检索 Chroma 相似案例
6. Sidecar 调用硅基流动分类
7. Rust 执行策略安全校验
8. 需要人工确认则返回 UI 弹窗
9. 确认后 Rust 执行移动或重命名
10. Rust 更新 SQLite
11. Rust 触发 Sidecar 写入高质量 RAG 样本

### 5.2 搜索链路

1. UI 发起搜索词
2. Rust 查询 SQLite `search_index`
3. Rust 组装搜索结果详情
4. UI 展示结果与本地路径
5. 用户可执行打开文件、定位目录、复制路径

### 5.3 撤销链路

1. UI 选择某次操作
2. Rust 校验当前文件状态与目标路径
3. Rust 执行逆向移动或逆向重命名
4. Rust 更新 `operations`、`managed_items`、`rag_samples`
5. UI 刷新历史和当前路径

## 6. 错误处理边界

### 6.1 UI 层错误

1. 输入为空
2. 搜索无结果
3. 人工确认内容不合法

处理方式：

1. 前端提示，不重试
2. 保留当前页面状态

### 6.2 Rust 层错误

1. 路径越界
2. 分类不存在
3. 文件系统执行失败
4. SQLite 写入失败
5. Sidecar 调用失败

处理方式：

1. 记录错误到日志与 `tasks.error_message`
2. 将任务置为 `failed` 或 `awaiting_review`
3. 禁止跳过策略层直接执行

### 6.3 Sidecar 错误

1. 解析失败
2. JSON-RPC 方法失败
3. 硅基流动超时或限流
4. Chroma 写入失败

处理方式：

1. 返回结构化错误码
2. 允许 Rust 根据错误码执行有限重试
3. 达到重试上限后回写任务失败

## 7. 配置边界

配置归 Rust 持有并写入 SQLite `settings` 表。Sidecar 不直接持久化配置，只接收 Rust 下发的运行时配置快照。

必须可配置项：

1. 目标根目录
2. 未分类目录
3. 暂存目录
4. 硅基流动 API Key
5. 聊天模型名
6. Embedding 模型名
7. 请求超时
8. 最大重试次数
9. 内容截断长度
10. 低置信度阈值

## 8. 架构不变式

以下不变式必须在实现中保持成立：

1. 文件系统写操作只能由 Rust 发起。
2. 搜索结果的路径展示必须来自 SQLite 中当前生效的 `managed_items.current_path`。
3. RAG 样本只有在满足质量门槛时才可写入 Chroma。
4. 任一重大变更都必须先修改 `dev_docs`。
