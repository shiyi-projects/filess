# UI 规范

## 1. 设计目标

首版 UI 目标是"拖入即整理、结果即可见"。用户关心的是文件去了哪里，而不是处理过程的技术细节。界面围绕"拖拽 → 查看结果路径 → 偶尔确认 → 偶尔搜索"组织。

采用暗色主题，视觉层次清晰，交互路径最短化。

## 2. 主界面信息架构

应用主结构为 `图标导航栏 + 主内容区 + 可折叠侧边栏` 的单屏布局。

三个一级入口通过导航栏始终可达：

1. `工作台`（主视图，默认展示）
2. `搜索`（Command Palette 浮层，快捷键 `Ctrl+K`）
3. `设置`（侧边抽屉面板）

### 2.1 图标导航栏

位于窗口左侧，宽度固定 48px。

必须包含以下图标入口：

1. 首页（工作台）
2. 搜索（触发 Command Palette）
3. 设置（触发设置抽屉）

### 2.2 工作台

工作台为默认主视图，从上到下分为三个区域：

1. 拖拽入口区
2. 整理结果区
3. 底部统计栏

### 2.3 可折叠活动侧边栏

位于主内容区右侧，默认折叠，可展开查看处理队列详情。

展开时必须显示：

1. 当前处理中的任务列表
2. 每个任务的实时状态
3. 待确认任务的快捷入口

折叠时在主界面通过徽章数字提示待处理事项数量。

### 2.4 搜索（Command Palette）

搜索以全局浮层形式呈现，不占据独立页面。

浮层包含：

1. 搜索输入框
2. 实时匹配的结果列表

搜索结果必须支持：

1. 打开文件
2. 打开所在位置
3. 复制路径

### 2.5 设置（抽屉面板）

设置以右侧抽屉面板形式呈现，不占据独立页面。设置内容必须可编辑，不得为只读展示。

## 3. 页面与组件规范

### 3.1 拖拽入口

位于工作台顶部，视觉上最突出。

必须展示：

1. 可拖入文件或文件夹的提示
2. 当前待处理数量（如有）
3. 拖入后的即时反馈

交互要求：

1. 必须实现真正的 drag-over / drop 事件处理。
2. 拖入后立即创建批次，不额外弹确认框。
3. 非法路径在原地提示，不跳转页面。
4. 拖拽悬停时必须有视觉反馈（边框发光、背景变化）。

### 3.2 整理结果区

位于拖拽入口下方，占据主内容区的主要空间。

展示最近成功整理的文件列表，每条结果必须展示：

1. 文件名
2. 分类名称
3. 当前本地路径（来自 `managed_items.current_path`）
4. 整理时间

每条结果必须支持：

1. 打开文件
2. 打开所在位置
3. 复制路径
4. 撤销操作

空状态时展示引导文案。

### 3.3 底部统计栏

固定于主内容区底部。

必须展示：

1. 今日整理数量
2. 待确认数量
3. 总管理文件数量
4. Sidecar 连接状态

### 3.4 搜索框

行为固定：

1. 搜索范围仅限已纳入本工具管理的文件。
2. 搜索字段固定为文件名、分类、路径、内容摘要。
3. 通过 `Ctrl+K` 快捷键或点击导航栏搜索图标触发。
4. 输入搜索词后在浮层内实时展示结果列表。
5. 不直接跳系统资源管理器。

### 3.5 搜索结果列表

在 Command Palette 浮层内展示。

每条结果必须展示：

1. 标题
2. 分类名称
3. 当前本地路径
4. 摘要片段

每条结果必须支持：

1. 键盘上下方向键选择
2. Enter 打开文件
3. Ctrl+Enter 打开所在位置
4. 快捷键复制路径

### 3.6 人工确认弹窗

触发条件固定为：

1. 低置信度
2. 未知类型
3. 重名冲突
4. 路径校验失败

可通过以下方式触发：

1. 活动侧边栏中点击待确认任务
2. 底部统计栏中点击待确认数字
3. 系统自动弹出（当任务进入 awaiting_review 状态时）

弹窗必须包含：

1. 文件名与原始路径
2. AI 建议分类
3. AI 建议名称
4. 置信度指示（可视化）
5. AI 给出的理由
6. 可编辑的分类选择器
7. 可编辑的名称输入框
8. 行为按钮

行为按钮固定为：

1. 确认分类
2. 修改名称
3. 改投未分类
4. 跳过

### 3.7 设置抽屉

设置抽屉从右侧滑入，不做插件开发中心。

分组固定为：

1. 基础路径
2. 分类规则
3. AI 设置
4. 整理策略
5. 数据与日志

每组最低字段要求：

1. 基础路径
   目标根目录、未分类目录、暂存目录
2. 分类规则
   分类树编辑、禁用分类、文件夹浅采样开关说明
3. AI 设置
   硅基流动 API Key、聊天模型、Embedding 模型、超时、重试、内容截断长度
4. 整理策略
   低置信度阈值、冲突策略、是否预览优先
5. 数据与日志
   SQLite 路径、日志级别、历史保留时长

所有设置项必须可编辑并支持保存。

## 4. 文案与反馈规范

1. 状态文案必须使用用户可读的中文，例如"正在提取特征""等待人工确认""已完成整理"。
2. 错误提示必须说明失败原因和下一步动作。
3. 成功整理后必须明确显示新的本地路径。
4. "打开所在位置"必须使用系统资源管理器定位，而不是仅打开父目录。
5. 不得在 UI 中直接展示英文状态枚举值。

## 5. 状态文案映射

1. `queued` → 排队中
2. `sniffing` → 识别类型
3. `parsing` → 提取内容
4. `retrieving_context` → 检索相似案例
5. `calling_model` → AI 分析中
6. `awaiting_review` → 需要确认
7. `executing` → 正在移动
8. `completed` → 整理完成
9. `failed` → 处理失败
10. `rolled_back` → 已撤销
11. `skipped` → 已跳过

## 6. 视觉规范

1. 采用深色主题作为默认主题。
2. 使用毛玻璃效果和微妙渐变营造层次感。
3. 拖拽区域必须是视觉上最突出的元素。
4. 所有状态变化必须有过渡动画。
5. 使用现代字体（Inter 或系统 UI 字体栈）。

## 7. 不允许的 UI 偏离

1. 不得把主界面改成完整资源管理器型三栏或树状浏览器。
2. 不得移除搜索能力。
3. 不得隐藏本地路径。
4. 不得把设置扩展成插件市场或多人协作后台。
5. 不得跳过应用内结果展示，直接把搜索结果变成纯系统跳转列表。
6. 不得将处理过程细节作为主视图的核心内容。
