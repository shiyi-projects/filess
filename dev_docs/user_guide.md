# AI 文件整理工具用户使用说明

## 1. 这是什么？

**AI 文件整理工具**是一款帮你整理本地文件的桌面应用。

你只需要把文件或文件夹拖进应用，它会根据文件名、文件内容摘要、历史整理记录和你的分类设置，自动判断文件应该放到哪里，并整理到指定目录中。

它适合整理：

- 下载目录里的发票、合同、资料、截图等文件
- 工作文档、学习资料、项目文件夹
- PDF、文本、图片和常见办公文件
- 暂时无法判断的文件会进入“未分类”，方便后续手动处理

---

## 2. 基本使用流程

推荐按下面顺序开始使用：

1. 打开应用。
2. 进入 **设置**。
3. 填写 AI 模型 API Key。
4. 设置文件整理目录。
5. 设置常用分类。
6. 回到主界面，把文件或文件夹拖进去。
7. 等待处理完成。
8. 在结果列表中查看文件被整理到了哪里。

---

## 3. 初次使用前需要设置什么？

### 3.1 设置 AI API Key

应用需要调用 AI 模型来判断文件分类。

首次使用前，请先打开左侧的 **设置**，在 **AI 模型** 中填写你的 API Key。

当前应用面向 SiliconFlow / 硅基流动模型服务使用。

在设置中你可以配置：

- API Key
- 对话模型
- 嵌入模型

如果没有配置可用的 API Key，自动分类和语义搜索可能无法正常工作。

### 3.2 设置整理目录

在 **设置 → 路径** 中，可以设置：

- **整理目录**：文件最终归档的位置
- **未分类目录**：AI 无法可靠判断时的兜底位置

例如，你可以把整理目录设置为：

```text
D:\Files
```

那么文件可能会被整理到：

```text
D:\Files\财务\发票\2026-03 供应商发票.pdf
D:\Files\工作\项目资料\需求文档.docx
D:\Files\学习\英语\听力资料.pdf
```

无法判断的文件会进入类似目录：

```text
D:\Files\未分类
```

### 3.3 设置常用分类

在 **设置 → 顶级分类** 中，每行填写一个分类。

例如：

```text
财务
工作
生活
学习
媒体
开发
```

AI 会优先根据这些分类来整理文件，也会结合历史记录生成更细的子分类。

例如：

```text
财务/发票
工作/客户资料
学习/课程讲义
开发/Rust
```

## 4. 如何整理文件？

### 4.1 拖拽文件或文件夹

在主界面顶部可以看到拖拽区域。

把文件或文件夹拖到这里，当界面提示 **“松开以开始整理”** 时，松开鼠标即可。

支持：

- 单个文件
- 多个文件
- 文件夹
- 批量拖入

拖入后，应用会自动开始处理，不需要额外确认。

### 4.2 手动选择文件

如果不方便拖拽，可以点击拖拽区域中的 **“或选择文件”**，从系统文件选择窗口中选择要整理的文件。

---

## 5. 处理过程中会看到什么？

拖入文件后，主界面会显示处理进度。

常见状态如下：

| 状态 | 含义 |
| --- | --- |
| 排队中 | 文件已加入队列，等待处理 |
| 嗅探 / 识别类型 | 正在判断文件类型 |
| 解析 / 提取内容 | 正在读取文件信息或内容摘要 |
| AI 分类 | 正在让 AI 判断文件分类 |
| 归档 | 正在移动或复制文件 |
| 完成 | 文件已整理完成 |
| 失败 | 处理过程中出错 |
| 等待审阅 | 需要你人工确认 |

右侧活动栏可以展开，查看当前正在处理的任务。

---

## 6. 查看整理结果

文件处理完成后，会出现在主界面的 **整理结果** 列表中。

每条结果会显示：

- 文件名
- 分类名称
- 当前本地路径
- 整理时间

例如：

```text
2026-03 供应商发票.pdf
分类：财务/发票
路径：D:\Files\财务\发票\2026-03 供应商发票.pdf
```

你最需要关注的是 **路径**，它告诉你文件现在被放到了哪里。

---

## 7. 整理结果中的常用操作

每条整理结果都提供几个快捷操作。

### 7.1 打开文件

点击结果卡片，或点击 **打开文件** 按钮。

应用会用系统默认程序打开该文件。

### 7.2 打开所在位置

点击 **打开所在位置**。

应用会打开系统资源管理器，并定位到该文件。

### 7.3 复制路径

点击 **复制路径**。

文件当前路径会被复制到剪贴板。

### 7.4 重新分类

如果你觉得 AI 分错了，可以点击 **重新分类**。

应用会让 AI 再判断一次。

你也可以输入一句提示帮助 AI 改正，例如：

```text
这是工作文件
```

或：

```text
归到 财务/报销
```

如果不想写提示，也可以留空直接重新分类。

### 7.5 撤销整理

如果你不想保留这次整理结果，可以点击 **撤销整理**。

确认后，应用会尝试把文件移回原来的位置。

注意：

- 如果原位置已经有同名文件，应用不会覆盖它。
- 应用会自动生成类似 `文件名_undo_1.pdf` 的名字，避免误覆盖。
- 如果文件已经被你手动移动或删除，应用会清理这条失效记录，并提示文件已不在记录位置。

## 8. 搜索文件

### 8.1 打开搜索

你可以通过两种方式打开搜索：

1. 点击左侧导航栏中的搜索图标。
2. 使用快捷键：

```text
Ctrl + K
```

快捷键可以在设置中修改。

### 8.2 搜索范围

搜索只会查找 **已经由本工具整理或纳入管理的文件**。

它不会扫描整个电脑，也不会搜索没有拖入过应用的文件。

### 8.3 搜索方式

搜索结果可能分为两类：

- **相关内容**：根据内容含义搜索
- **文件名匹配**：根据文件名、分类、路径等信息搜索

例如你可以搜索：

```text
发票
合同
客户资料
英语听力
项目计划
```

### 8.4 搜索快捷键

在搜索窗口中可以使用：

| 快捷键 | 功能 |
| --- | --- |
| ↑ / ↓ | 选择结果 |
| Enter | 打开文件 |
| Ctrl + Enter | 打开所在位置 |
| Ctrl + C | 复制路径 |
| Esc | 关闭搜索 |

## 9. 设置项说明

### 9.1 AI 模型

用于设置 AI 分类和语义搜索需要的模型信息。

主要包括：

- API Key
- 对话模型
- 嵌入模型

### 9.2 性能

可以设置 **并发处理任务数**。

数值越大，同时处理的文件越多，但也可能更容易遇到 API 限流。

一般建议保持应用推荐值，通常为 3 到 5 左右。

### 9.3 快捷键

可以设置快速搜索快捷键。

默认是：

```text
Ctrl + K
```

建议使用带有 Ctrl 或 Alt 的组合键，避免和普通输入冲突。

### 9.4 路径

可以设置：

- 整理目录
- 未分类目录

### 9.5 顶级分类

每行填写一个分类，例如：

```text
财务
工作
生活
学习
媒体
开发
```

### 9.6 策略

可以设置：

- 低置信度阈值
- 顶级分类上限

低置信度阈值越高，AI 越谨慎，更多文件可能会进入未分类。

## 10. 移动、复制和源文件

应用支持不同的整理方式。

### 10.1 移动模式

移动模式下，应用会把文件整理到目标目录，并处理原位置的文件。

通常更适合正式整理文件。

### 10.2 复制模式

复制模式下，应用会复制一份文件到整理目录，原文件保留不动。

如果你是第一次使用，建议先用复制模式试一批文件，确认分类效果后再进行大量整理。

### 10.3 源文件处理

在移动模式下，源文件可以被移入回收站或删除。

为了安全，建议优先使用回收站方式。

---

## 11. 重名文件会怎样处理？

如果目标目录中已经存在同名文件，应用不会覆盖已有文件。

它会自动加序号，例如：

```text
合同.pdf
合同_1.pdf
合同_2.pdf
```

这样可以避免误覆盖重要文件。

---

## 12. 重复文件会怎样处理？

应用会识别已经整理过的相同文件。

如果你再次拖入同一个文件，应用通常会跳过重复整理，并把已有记录显示到最近结果中，方便你查看它现在的位置。

---

## 13. 文件夹整理说明

你可以直接拖入文件夹。

应用会根据文件夹名称和其中部分内容进行判断。

需要注意：

- 文件夹会保持原文件夹名称。
- 文件夹不会像单个文件那样被 AI 改名。
- 文件夹内的内容会随文件夹一起移动或复制。

---

## 14. 隐私与安全说明

为了安全，应用遵循以下规则：

1. AI 只给出分类建议，不直接控制完整物理路径。
2. 最终文件路径由本地应用根据整理目录和分类规则生成。
3. 默认不会覆盖已有文件。
4. 搜索范围仅限已经纳入管理的文件，不会全盘扫描。
5. 发送给 AI 的是必要的文本片段和摘要，不是完整文件。
6. 整理结果和历史记录保存在本地数据库中。

如果你要整理敏感文件，请先确认是否允许向云端模型发送必要的文本片段。

## 15. 常见问题

### Q1：为什么文件进了“未分类”？

可能原因包括：

- AI 判断置信度较低
- 文件类型无法可靠解析
- 分类结果不安全
- 当前分类规则不够明确
- 顶级分类数量达到上限

你可以点击 **重新分类**，并输入提示帮助 AI 判断。

### Q2：为什么搜索不到某个文件？

搜索只查找已经由本工具管理的文件。

如果某个文件从未拖入应用整理，就不会出现在搜索结果中。

### Q3：为什么再次拖入文件没有重新整理？

应用可能识别到这是已经整理过的重复文件，因此跳过了重复处理。

你可以在最近结果或搜索中查看它已有的位置。

### Q4：撤销后文件名为什么变了？

如果原位置已经有同名文件，应用不会覆盖它。

因此会自动生成类似下面的名字：

```text
文件名_undo_1.pdf
```

### Q5：AI 能不能直接把文件放到我指定的完整路径？

不能。

为了安全，AI 只能建议分类，不能直接指定完整磁盘路径。

实际路径由本地应用生成。

### Q6：应用会自动扫描整个电脑吗？

不会。

当前版本只处理你主动拖入或选择的文件和文件夹。

### Q7：语义搜索为什么有时比较慢？

语义搜索需要调用嵌入模型理解搜索词含义，可能需要等待网络请求。

文件名匹配通常会更快。

## 16. 推荐使用方式

如果你是第一次使用，建议这样操作：

1. 先配置 API Key。
2. 设置整理目录和未分类目录。
3. 设置几个常用分类。
4. 先选择复制模式。
5. 拖入少量测试文件。
6. 检查整理结果是否符合预期。
7. 如果分类不满意，使用重新分类并补充提示。
8. 确认效果后，再批量整理更多文件。
9. 平时通过 `Ctrl + K` 快速搜索已经整理过的文件。

---

## 17. 一句话总结

把文件拖进去，AI 会帮你判断分类并整理到指定目录；整理完成后，你可以查看文件去向、搜索、打开、定位、复制路径、撤销或重新分类。
